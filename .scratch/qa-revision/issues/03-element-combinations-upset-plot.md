# 03 — Replace the "Element combinations" dot-matrix with a proper UpSet plot

Status: ready-for-human
Category: enhancement

## What happened

The Dashboard's Minor/Major deck segments show an "Element combinations" view rendered as a hand-rolled dot-matrix (one row per element set, dots plus a count column). The reporter likes the tab overall, but the visual doesn't match the mindset of an UpSet plot — set sizes and intersections aren't visually comparable the way https://upset.js.org/app/ presents them.

## What I expected

An UpSet-style plot for element combinations: intersection-size bars over the combination dot matrix (and ideally per-element set-size bars), so the relative frequency of each element combination is readable at a glance.

## Steps to reproduce

1. Open Dashboard → Minor (or Major) deck segment.
2. Look at "Element combinations": counts are plain numbers next to dot rows, with no bar encoding of intersection sizes.

## Additional context

- Owner decision: **prototype both approaches first** before committing — (a) adopting the UpSet.js library (`@upsetjs/react`) vs (b) custom-building the UpSet layout on the existing hand-rolled JSX+CSS approach (the repo currently has zero charting dependencies; everything is dots and percentage-width bars styled via `deck-*` classes). Run through `/prototype`, keep the answer, then implement.
- The data layer already produces exactly what an UpSet plot needs (combination groups sorted by count, canonical element order), so this is expected to be a presentation-layer change only.
- Whatever wins should preserve the existing "Highlight my spirit" element-highlight behavior.
